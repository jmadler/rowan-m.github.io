I've been framed!

This is an AMP page which installs a service worker in the background. If you visit this page without an existing service worker then you will see the AMP content rendered as you would expect. Once the service worker is active, then loading this page will actually cause the shell to be returned in the response. The shell will then check its path and load the relevant AMP content. From an end user point of view, this means they will get upgraded to the shell experience.
